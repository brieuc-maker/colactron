# 02 - UX / UI

## Direction artistique

Interface premium, claire et professionnelle.

- Sidebar sombre.
- Contenu principal clair.
- Cards arrondies.
- Tableaux lisibles.
- Badges de statut colorés.
- Animations discrètes.
- Forte lisibilité des chiffres.

## Pages principales

### 1. Dashboard

Objectif : donner une vision instantanée du portefeuille.

Composants : KPI cards, graphiques, table derniers biens, alertes, prochaines actions.

### 2. Liste des biens

Objectif : consulter et filtrer tous les biens.

Vues : table, cards, kanban, carte.

Filtres : statut, ville, ROI, TRI, budget travaux, responsable, stratégie, PEB.

### 3. Fiche bien

Page centrale de l'application.

Structure : header du bien, photo principale, badge statut, actions rapides, onglets détaillés.

### 4. Pipeline

Vue Kanban drag & drop.

Chaque colonne représente une étape du cycle de vie.

### 5. Visites

Vue calendrier + liste. Une visite doit pouvoir être ouverte en plein écran pour encoder le compte rendu.

### 6. Travaux

Vue Gantt + Kanban + tableau budgétaire.

### 7. Finances

Vue portefeuille + fiche financière par bien.

### 8. Documents

Gestion documentaire avec arborescence, preview PDF, OCR et recherche.

### 9. Carte

Carte interactive affichant tous les biens par statut et rentabilité.

### 10. Paramètres

Gestion utilisateurs, société, rôles, sources, IA, intégrations.

## Etats UI obligatoires

Chaque écran doit prévoir :

- Etat vide.
- Chargement.
- Erreur.
- Données partielles.
- Accès refusé.
- Confirmation avant suppression.

## Responsive

Priorité desktop. Tablette secondaire. Mobile limité aux vues essentielles : dashboard, fiche bien, visite, documents.
